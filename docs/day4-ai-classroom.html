Day 4 Summary

오늘의 핵심 목표

Day 4에서는 화면 설계, 사용자 흐름, 데이터 구조를 바탕으로 실제 서비스 기능을 API 형태로 정의하고, Swagger/OpenAPI 형식의 명세서로 문서화합니다.

  • API 엔드포인트 도출: 화면 요구사항 기반 REST API 리소스 및 URI 정의
  • Request/Response 규격 설계: JSON 구조의 요청/응답 페이로드 스키마 정의
  • HTTP 메소드 및 상태 코드 정의: GET, POST, PUT, DELETE 규칙과 에러 처리 규격 설계
  • Swagger/OpenAPI 명세 작성: YAML/JSON 형식을 이용한 명세서 작성 기법 습득
  • 개발 인수인계(Handoff) 고려: 예외 응답 코드와 에러 메시지 세부화
학습 팁: 오른쪽 AI 챗봇에게 “RESTful API 설계의 기본 규칙이 뭐야?” 또는 “Swagger에서 Request Body와 Path Parameter를 어떻게 명시해?” 같은 질문을 해보세요.
