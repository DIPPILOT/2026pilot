Day 1 Detailed Curriculum

AI Product Wireframing with Figma

Figma 입문자를 대상으로, User Flow를 기반으로 한 서비스 화면 구조 설계와 핵심 화면 와이어프레임 제작, 프로토타입 연결, 협업 기능 활용까지 경험하는 1일차 학습 모듈입니다.

학습 개요

Day 1은 서비스 아이디어를 실제 제품 화면 구조로 전환하는 첫 단계입니다. 참가자는 Figma의 기본 사용법을 익히고, Landing / Dashboard / Auth와 같은 핵심 화면의 구조를 설계합니다.

단순히 화면을 그리는 수준을 넘어, 사용자 흐름(User Flow)을 기준으로 서비스의 주요 진입점과 화면 간 연결 관계를 이해하고 프로토타입으로 이어지는 기초 설계 과정을 실습합니다.

주요 세션 구성

Session 1 · 2시간
Figma 사용법 및 화면 설계 방법론 이해

Figma의 기본 인터페이스와 핵심 도구를 익히고, 서비스 화면을 어떤 논리로 구조화할지 이해합니다.

  • 프레임, 텍스트, 도형, 오토레이아웃 기초
  • 서비스 화면 설계의 기본 원칙
  • User Flow와 화면 구조의 관계 이해
Session 2 · 3시간

Figma 화면 설계를 기반으로 사용자 행동과 시스템 상호작용을 시간 순서대로 시각화하고, 데이터 흐름과 예외 상황을 함께 설계하는 2일차 학습 모듈입니다.

학습 개요

Day 2는 전날 설계한 화면 구조를 실제 시스템 동작 흐름으로 확장하는 단계입니다. 참가자는 화면 위에서 보이는 UI를 넘어서, 사용자의 행동이 어떤 순서로 서버와 데이터베이스에 전달되고 처리되는지를 이해하게 됩니다.

이를 위해 시퀀스 다이어그램을 활용해 사용자, 클라이언트, 서버, 데이터베이스 간의 상호작용을 시간 순서대로 정리하고, 정상 흐름뿐 아니라 오류와 실패 상황까지 함께 설계합니다.

Day 3 Detailed Curriculum

Data Normalization & ERD Design

화면 설계와 사용자 흐름을 기반으로 필요한 데이터를 구조화하고, 핵심 엔티티와 관계를 정의하여 정규화된 데이터 구조와 ERD를 완성하는 3일차 학습 모듈입니다.

학습 개요

Day 3는 앞선 화면 설계와 데이터 흐름 설계를 실제 데이터베이스 구조로 연결하는 단계입니다. 참가자는 화면과 사용자 흐름에서 필요한 데이터를 추출하고, 이를 엔티티와 속성 단위로 정리하게 됩니다.

이후 엔티티 간 관계를 정의하고, 1NF부터 3NF까지의 정규화 과정을 적용하여 중복을 줄이고 무결성을 높인 데이터 구조를 설계합니다. 최종적으로는 서비스 구현에 활용 가능한 ERD를 완성합니다.

주요 세션 구성

Session 1 · 2시간
ERD 설계 이해 및 데이터 추출

화면과 사용자 흐름을 기준으로 어떤 데이터가 필요한지 파악하고, 이를 구조화 가능한 데이터 항목으로 정리합니다.

  • 스크린과 유저 플로우 기반 데이터 추출
  • 데이터 필드 정의 및 그룹화
  • ERD의 개념과 목적 이해
Session 2 · 3시간
엔티티/관계 정의 및 정규화 실습

추출한 데이터를 엔티티로 전환하고, 핵심 관계를 정의한 뒤 정규화 과정을 통해 효율적인 데이터 구조를 설계합니다.

  • 핵심 엔티티와 속성 정의
  • 1:1, 1:N, N:M 관계 설정
  • 1NF → 2NF → 3NF 정규화 적용
Session 3 · 1시간
ERD 검토 및 결과 공유

완성한 데이터 구조와 ERD를 공유하고, 중복 제거와 관계 설정의 적절성을 함께 검토합니다.

  • ERD 다이어그램 발표
  • 정규화 결과 리뷰
  • 피드백 기반 수정 포인트 정리

Day 10 Detailed Curriculum

Deployment & Operation

Docker 컨테이너화, GitHub Actions 기반 CI/CD 자동화, 그리고 클라우드 서버 배포까지 로컬 서비스를 실제 운영 환경으로 연결하는 10일차 학습 모듈입니다.

학습 개요

Day 10은 지금까지 만든 제품을 로컬 환경에서 끝내지 않고 실제 운영 환경으로 배포하는 단계입니다. 참가자는 실행 환경을 Docker로 통일하고, 코드 변경이 자동으로 배포까지 이어지는 CI/CD 흐름을 이해합니다.

또한 AWS/NCP 같은 클라우드 환경에 서비스를 배포하고, 환경 변수, 보안 설정, 도메인, SSL 같은 운영 요소까지 함께 점검하여 MVP를 실제 외부에 공개할 수 있는 수준으로 완성합니다.

주요 세션 구성

Session 1 · 1시간
배포 구조와 Docker 이해

로컬 실행과 운영 배포의 차이를 이해하고, 컨테이너 기반 환경 통일의 필요성을 학습합니다.

  • 배포 아키텍처 기본 개념 이해
  • Docker 컨테이너화와 환경 일치
  • 배포 체크리스트와 사전 점검 항목 확인
Session 2 · 1시간
CI/CD 자동화 및 클라우드 배포 실습

Docker 실행과 GitHub Actions 설정을 통해 배포 자동화를 구성합니다.

  • Dockerfile 작성 및 docker run 실행
  • GitHub Actions workflow 설정
  • 클라우드 서버 배포와 환경 변수 관리
Session 3 · 1시간
배포 결과 확인 및 장애 대응 공유

최종 배포 상태를 확인하고 운영 과정에서 발생할 수 있는 이슈를 검토합니다.

  • 배포 결과 점검 및 접속 확인
  • 도메인 / SSL / 보안 설정 검토
  • 배포 실패 사례와 트러블슈팅 공유
